## Методология моделирования

### Типы моделирования

| Тип | Направление | Применение |
|-----|-------------|------------|
| Восходящее | L3 → L2, L2 → L1 | Когда есть данные нижнего уровня |
| Нисходящее | L1 → L2 | Когда есть стратегические цели верхнего уровня |
| Горизонтальное | L3 взаимодействия | Для понимания операционных связей |

### Принципы

1. **Прослеживаемость:** Каждая цель/задача/процесс имеет поле `source` с указанием источника
2. **Связанность:** Цели L2 привязаны к целям L1 через поле `linked_department_goal`
3. **Валидация взаимодействий:** Процессы включают `interactions_from_L3` с цитатами из анкет

### Структура JSON-файлов

```
L1 (Департамент):
├── goals[]
├── tasks[] → привязаны к goals
└── processes[] → привязаны к tasks

L2 (Отдел):
├── division_goals[] → связаны с L1 goals
└── division_tasks[]
    └── task_processes[] → включают interactions_from_L3
```
