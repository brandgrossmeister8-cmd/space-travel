**✅ Фактические данные (из анкет L3):**

### Формула расчёта

```
часы/мес = time_minutes × множитель_частоты ÷ 60

Множители: Ежедневно = 22, Еженедельно = 4.33, Ежемесячно = 1, По запросу = 5
```

**Пример:** Антошкина, "Корректировки в заявку", 500 мин, ежедневно → 500 × 22 ÷ 60 = **183 ч/мес**

### Что такое FTE

**FTE** (Full-Time Equivalent) = эквивалент полной занятости = способ перевести часы в "количество сотрудников".

```
1 сотрудник = 22 дня × 8 часов = 176 часов/месяц
FTE = Общие часы ÷ 176
```

## Глоссарий терминов (для неспециалистов)

> **Все термины объяснены простым языком для тех, кто не работает в IT или туризме.**

### Общие бизнес-термины

| Термин | Расшифровка | Простое объяснение |
|--------|-------------|-------------------|
| **FTE** | Full-Time Equivalent (Эквивалент полной ставки) | Способ измерения нагрузки: 1 FTE = работа одного человека 8 часов в день. Если задача занимает 88 ч/мес — это 0.5 FTE (полставки). |
| **ROI** | Return on Investment (Возврат инвестиций) | Показывает, сколько денег вернётся от вложений. ROI 200% = вложили 1 рубль, получили 2 рубля прибыли. |
| **KPI** | Key Performance Indicators (Ключевые показатели) | Цифры, по которым оценивают результат работы. Например: количество продаж, скорость ответа клиенту. |
| **Quick Wins** | Быстрые победы | Задачи, которые легко сделать и быстро получить результат. Начинают с них, чтобы показать эффект. |
| **Bottleneck** | Узкое место | Этап процесса, где всё «застревает». Как пробка на дороге — весь поток замедляется из-за одного места. |
| **AS-IS** | Как есть сейчас | Описание того, как процесс работает в данный момент (со всеми проблемами). |
| **TO-BE** | Как должно быть | Описание целевого состояния — как процесс будет работать после улучшений. |
| **Roadmap** | Дорожная карта | План внедрения по шагам: что делаем сначала, что потом, в какие сроки. |
| **MVP** | Minimum Viable Product (Минимально работающий продукт) | Первая версия системы с базовым функционалом — чтобы быстрее запустить и проверить, работает ли идея. |
| **NET** | Чистый показатель | Итоговое значение после вычета всех дублей и пересечений. NET-экономия = реальная экономия. |

### IT и автоматизация

| Термин | Расшифровка | Простое объяснение |
|--------|-------------|-------------------|
| **API** | Application Programming Interface (Программный интерфейс) | Способ, которым одна программа «разговаривает» с другой автоматически. Как электронный переводчик между системами. |
| **Интеграция** | Связка систем | Когда две программы обмениваются данными автоматически, без ручного ввода. |
| **RPA** | Robotic Process Automation (Роботизация процессов) | Программный робот, который делает рутинную работу вместо человека: заполняет формы, копирует данные, нажимает кнопки. |
| **iPaaS** | Integration Platform as a Service (Платформа интеграций) | Сервис для соединения разных программ без написания сложного кода. Как конструктор LEGO для IT-систем. |
| **Low-code** | Мало кода | Платформа, где можно создавать программы почти без программирования — через визуальный редактор с блоками. |
| **BI** | Business Intelligence (Бизнес-аналитика) | Система для создания отчётов и графиков из данных. Показывает картину бизнеса: продажи, доходы, тренды. |
| **Дашборд** | Dashboard (Панель показателей) | Экран с ключевыми цифрами и графиками в реальном времени. Как приборная панель в машине — видно всё важное сразу. |
| **CRM** | Customer Relationship Management (Управление клиентами) | Система, где хранится вся информация о клиентах: контакты, история покупок, переписка. У нас — Битрикс24. |
| **ERP** | Enterprise Resource Planning (Управление ресурсами) | Большая система для управления всей компанией: финансы, склад, персонал, производство. У нас — 1С. |
| **SLA** | Service Level Agreement (Уровень сервиса) | Договорённость о качестве услуги: например, отвечать на заявки не дольше 2 часов. |
| **LLM** | Large Language Model (Большая языковая модель) | Искусственный интеллект, который понимает и генерирует текст. Например, ChatGPT, Claude. |
| **ML** | Machine Learning (Машинное обучение) | Технология, когда компьютер сам учится на данных и делает прогнозы. Например, предсказывает спрос на туры. |

### Туристическая отрасль

| Термин | Расшифровка | Простое объяснение |
|--------|-------------|-------------------|
| **GDS** | Global Distribution System (Глобальная система бронирования) | Международная система для бронирования авиабилетов. Amadeus, Galileo, Sabre — основные игроки. |
| **MICE** | Meetings, Incentives, Conferences, Events | Корпоративный туризм: организация конференций, командировок, корпоративных поездок для компаний. |
| **DMC** | Destination Management Company (Принимающая компания) | Партнёр на курорте, который организует услуги на месте: трансферы, экскурсии, гостиницы. |
| **Чартер** | Charter (Чартерный рейс) | Самолёт, арендованный туроператором целиком. Билеты дешевле, но расписание негибкое. |
| **Тайм-лимит** | Time Limit (Срок оплаты) | Дедлайн, до которого нужно оплатить забронированный билет, иначе бронь отменяется. |
| **Турпродукт** | Туристический продукт | Готовый пакет для туриста: перелёт + отель + трансфер + экскурсии. |
| **Lead-to-Cash** | От заявки до денег | Весь путь клиента: от первого обращения до получения оплаты. Главный процесс продаж. |
| **ЕФРТА** | Единый федеральный реестр турагентов | Государственный реестр, где должны быть зарегистрированы все турагенты России. |
| **Турпомощь** | Объединение туроператоров | Организация, которая помогает туристам при банкротстве туроператора. Взносы обязательны. |

### Финансовые термины

| Термин | Расшифровка | Простое объяснение |
|--------|-------------|-------------------|
| **P&L** | Profit and Loss (Прибыли и убытки) | Отчёт, показывающий доходы минус расходы = прибыль (или убыток). |
| **Cash Flow** | Денежный поток | Движение денег: сколько пришло, сколько ушло, сколько осталось на счетах. |
| **Маржа** | Margin (Наценка) | Разница между ценой продажи и себестоимостью. Маржа 10% = с каждых 100 рублей остаётся 10 рублей. |
| **Платёжный календарь** | Payment Calendar | График: когда и сколько нужно заплатить поставщикам, когда придут деньги от клиентов. |

### Управление проектами

| Термин | Расшифровка | Простое объяснение |
|--------|-------------|-------------------|
| **Workflow** | Рабочий процесс | Последовательность шагов для выполнения задачи: кто делает, что передаёт, кому. |
| **SMART** | Specific, Measurable, Achievable, Relevant, Time-bound | Правило постановки целей: конкретная, измеримая, достижимая, актуальная, со сроком. |
| **ТЗ** | Техническое задание | Документ, описывающий что нужно сделать: требования, сроки, бюджет, результат. |
| **AUTO-точка** | Точка автоматизации | Конкретная задача или процесс, который можно автоматизировать. У нас 62 таких точки. |
| **Волна внедрения** | Implementation Wave | Группа задач, которые внедряются одновременно. Волна 1 — самые простые и быстрые. |
| **Gantt** | Диаграмма Ганта | Визуальный график проекта: горизонтальные полоски показывают, когда что делается. |
