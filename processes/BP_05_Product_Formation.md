## 3. Ключевые цели процесса

1. **Формирование продуктового портфеля** — создание конкурентных туров по всем направлениям
2. **Обеспечение жизнедеятельности продукта** — мониторинг конверсии, актуализация цен
3. **Увеличение операционных доходов** — работа с наценкой, incentives, kick-backs
4. **Конкурентное ценообразование** — поддержание цен на уровне или ниже конкурентов

### 4.3. Ценообразование чартерного продукта

1. Мониторинг загрузки рейсов

- 🔴 Полностью ручной процесс корректировки
- ⚠️ Задержки в реакции на рынок

### 4.5. Выявление и устранение ошибок

**Типы ошибок:**

| Тип | Источник | Процесс устранения |
|-----|----------|-------------------|
| **Тарификационная** | Неоднозначные формулировки, ошибка в Excel | Группа тарификации → исправление → анализ причин |
| **Техническая** | Ошибка САМО-софт, интеграции | Группа тарификации → IT → САМО-софт |

**Алгоритм:**
1. Поступление задачи (Битрикс/почта) с описанием ошибки
2. Определение типа ошибки
3. Исправление / делегирование в IT
4. Анализ затронутых бронирований
5. Минимизация финансовых потерь (подключение групп стран)
6. Отчёт руководителю

## 7. Риски и проблемы

### Критические

| Проблема | Причина | Влияние |
|----------|---------|---------|
| **Группа тарификации — узкое место** | 5 человек на ВСЕ цены компании | Задержки публикации 2-5 недель |
| **Ручное ценообразование чартеров** | Нет автоматизации | Потеря маржи, горение мест |

### Высокие

| Проблема | Причина | Влияние |
|----------|---------|---------|
| Длительный сбор информации | Несистематизированные данные регионов | Затягивание планирования |
| Троение гостиниц от динамических поставщиков | Нет автоматического сопоставления | Ручная обработка |
| Отсутствие учёта обращений | Обращения по почте, не в CRM | Нет данных о нагрузке |

## 10. Рекомендации

### Немедленные действия

1. **Внедрить динамическое ценообразование чартеров** — критично для снижения убытков
2. **Автоматизировать мониторинг сроков действия цен** — предотвращение пустых сезонов
3. **Настроить парсинг обращений из почты** — получить метрики по нагрузке

### Среднесрочные

1. Автоматическое сопоставление гостиниц от разных поставщиков
2. Робот для проверки ошибок в расчётах
3. Автоматическая ревизия туров без конверсии

### Долгосрочные

1. Единая система управления продуктом с интеграцией всех поставщиков
2. Предиктивная аналитика спроса для планирования продукта
3. Автоматическое формирование продуктового плана на основе данных
