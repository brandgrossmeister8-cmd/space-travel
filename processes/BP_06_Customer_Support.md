3. Внесение в инвойс-трекер (СЗ или СБ)
4. Передача в бухгалтерию

**Ошибки:** Путаница в способах оплаты, внесение в неверный трекер

### 4.6. Поддержка MICE-клиентов (корпоративный сегмент)

| Параметр | Значение |
|----------|----------|
| **Исполнитель** | Отдел круглосуточной работы с клиентами |
| **Связанный процесс** | BP-03 (MICE и корпоративное обслуживание) |
| **Режим** | 24/7 (в нерабочее время Отдела группового бронирования) |
| **Критичность** | 🔴 КРИТИЧНО для удержания корпоративных клиентов |

**Почему это критично:**
- Корпоративные клиенты (B2B) составляют **~10% выручки** и имеют **высокие ожидания по уровню сервиса**
- MICE-мероприятия (конференции, групповые поездки) часто проходят **в нестандартное время**
- Потеря доверия корпоративного клиента = **потеря долгосрочного контракта** (средний чек в 5-10 раз выше B2C)
- Отсутствие 24/7 поддержки — **конкурентный недостаток** на рынке MICE

**Алгоритм обработки MICE-обращений:**

```
    ТРИГГЕР: Обращение от MICE-клиента (телефон / мессенджер / email)
                         │
                         ▼
    ┌─────────────────────────────────────────────────────┐
    │  1. Идентификация MICE-клиента                      │
    │  - Проверка тега «MICE» / «Корпоративный» в Битрикс │
    │  - Проверка номера заявки в САМО (тип = групповая)  │
    └─────────────────────────┬───────────────────────────┘
                              │
                              ▼
    ┌─────────────────────────────────────────────────────┐
    │  2. Создание сделки «MICE-поддержка 24/7» в Битрикс │
    │  - Номер заявки САМО                                │
    │  - Название компании-клиента                        │
    │  - Суть обращения                                   │
    │  - Контакты представителя                           │
    └─────────────────────────┬───────────────────────────┘
                              │
                              ▼
    ┌─────────────────────────────────────────────────────┐
    │  3. Приоритетная обработка                          │
    │  - SLA: ответ <=15 мин, решение <=2 часа            │
    │  - Связь с DMC / партнёром / отелем                 │
    │  - При необходимости: эскалация на руководителя     │
```

## 7. Риски и проблемы

### Критические

| Проблема | Причина | Влияние | Статус |
|----------|---------|---------|--------|
| **28 направлений маршрутизации вручную** | Нет автоматической маршрутизации | Задержки, ошибки | 🔴 ОТКРЫТО |

### Высокие

| Проблема | Причина | Влияние |
|----------|---------|---------|
| Долгое ожидание ответа Отдела продаж | Высокая загрузка | Пропуск звонков |
| Отсутствия сотрудников в разных системах | Нет единой платформы | Задержки маршрутизации |
| Не все агенты в САМО/Битрикс | Неполные данные | Поиск в интернете |
| Устаревшие списки направлений | Не обновляются | Неверная маршрутизация |
| Ручной перенос данных САМО → Битрикс | Нет интеграции | Дублирование работы |

### Средние

| Проблема | Причина | Влияние |
|----------|---------|---------|
| Функционал ГП, заявлений не по профилю | Исторически сложилось | Отвлечение от основных задач |
| Большое время на прослушку звонков | Ручной контроль качества | Нехватка времени руководителя |

## 8.1. Детализация AUTO-SUP-03: ИИ-анализ качества звонков

### Описание

Автоматизированная система анализа качества звонков с использованием ИИ (речевая аналитика) для оценки работы операторов клиентской поддержки и 24/7 службы.

### Критерии оценки качества звонков

| Критерий | Вес | Описание | Целевой показатель |
|----------|-----|----------|-------------------|
| **1. Приветствие** | 10% | Корректное приветствие по стандарту, представление себя и компании | 100% соблюдение |
| **2. Выявление потребности** | 25% | Уточняющие вопросы, активное слушание, понимание запроса клиента | ≥4 из 5 баллов |
| **3. Компетентность** | 25% | Знание продукта, корректность информации, владение системами | ≥4 из 5 баллов |
| **4. Решение вопроса** | 20% | Полнота ответа, предложение альтернатив, FCR (решение с первого обращения) | FCR ≥70% |
| **5. Завершение разговора** | 10% | Подведение итогов, уточнение «всё ли понятно», прощание | 100% соблюдение |
| **6. Время разговора** | 10% | Соответствие нормативам: простой вопрос ≤3 мин, сложный ≤7 мин | ≤норматива |

### Дополнительные параметры анализа

| Параметр | Описание | Красные флаги |
|----------|----------|---------------|
| **Тон и эмоции** | Анализ эмоциональной окраски диалога | Раздражение, повышение голоса |
| **Скорость речи** | Темп и внятность речи оператора | Слишком быстро / нечётко |
| **Паузы и «э-э-э»** | Заминки, неуверенность | >5 пауз, >3 сек молчания |
| **Запрещённые фразы** | «Я не знаю», «Это не ко мне», «Перезвоните» | Любое использование |
| **Перебивание клиента** | Прерывание клиента на полуслове | >2 раз за звонок |

### Формат передачи данных в HR

**Еженедельный отчёт (автоматический):**

| Раздел | Содержание |
|--------|------------|
| **Сводка по критериям** | Средние оценки по каждому из 6 критериев за неделю |
| **ТОП-3 проблемы** | Наиболее частые нарушения / низкие оценки |
| **Рейтинг сотрудников** | Оценки по каждому оператору (с динамикой) |
| **Красные флаги** | Сотрудники с оценкой <60% — требуют внимания |
| **Лучшие практики** | Примеры отличных звонков для обучения |

**Триггеры для немедленного уведомления HR:**
- Оценка сотрудника <50% по итогам 10+ звонков
- 3+ жалобы клиентов на одного сотрудника за неделю
- Грубое нарушение стандарта (использование запрещённых фраз, конфликт)

### Ожидаемый эффект интеграции

| Метрика | До интеграции | После интеграции |
|---------|---------------|------------------|
| Время руководителя на контроль | 8-10 ч/неделю | 1-2 ч/неделю |
| Охват анализа звонков | ~5% | 100% |
| Время выявления проблем | 2-4 недели | 1 неделя |
| Целевое обучение | Общее | Персонализированное |

## 10. Рекомендации

### Немедленные действия

1. ~~**Настроить связь MICE с 24/7 поддержкой**~~ — ✅ РЕШАЕТСЯ (см. раздел 4.6)
2. **Обновить списки направлений и сотрудников** — актуализировать маршрутизацию
3. **Внедрить единую платформу учёта отсутствий** — все в одном месте

### Среднесрочные

1. Автоматическая маршрутизация по региону (привязка к ФО)
2. Переадресация звонков регионов по номеру телефона
3. ИИ-анализ качества звонков

### Долгосрочные

1. Полная интеграция САМО ↔ Битрикс (карточки, сделки)
2. Омниканальная платформа для всех обращений
3. Чат-бот для первичной классификации обращений
