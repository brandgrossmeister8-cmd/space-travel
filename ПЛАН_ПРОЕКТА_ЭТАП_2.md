## ОБЗОР ПРОЕКТА

### Цель проекта
Провести анализ организационной структуры и бизнес-процессов Space Travel с фокусом на быстрое выявление точек автоматизации:
- Иерархия целей и задач (L1 → L2 → L3)
- Карта бизнес-процессов и узких мест
- Выявление точек автоматизации с расчётом ROI (окупаемости)
- ТЗ на автоматизацию и дорожная карта
- Регламенты и должностные инструкции

### Ключевые принципы
1. **Иерархичность**: Цель департамента → Задачи → Цель отдела → Задачи → Процессы → Функционал
2. **SMART-цели**: Конкретные, измеримые, достижимые, релевантные, ограниченные во времени
3. **Фокус на автоматизацию**: Приоритет задачам с высоким ROI
4. **Практичность**: Минимум бюрократии, максимум пользы для бизнеса

## РАСШИФРОВКА ТЕРМИНОВ

| Термин | Расшифровка |
|--------|-------------|
| **ROI** | Return on Investment — окупаемость инвестиций |
| **KPI** | Key Performance Indicators — ключевые показатели эффективности |
| **SMART** | Specific, Measurable, Achievable, Relevant, Time-bound — конкретная, измеримая, достижимая, релевантная, ограниченная во времени (цель) |
| **Bottleneck** | Узкое место — точка в процессе, где возникают задержки |
| **Quick Win** | Быстрая победа — инициатива с высоким эффектом и низкой сложностью |
| **AS-IS** | Текущее состояние (как есть сейчас) |
| **TO-BE** | Целевое состояние (как должно быть) |
| **Value Chain** | Цепочка создания ценности |
| **Cycle Time** | Время цикла процесса от начала до конца |
| **RPA** | Robotic Process Automation — роботизация процессов |
| **iPaaS** | Integration Platform as a Service — интеграционная платформа |
| **BI** | Business Intelligence — бизнес-аналитика |
| **L1, L2, L3** | Уровни управления: L1 — департаменты, L2 — отделы, L3 — сотрудники |

### Имеющиеся файлы

| Уровень | Количество | Описание |
|---------|------------|----------|
| L1 | 8 файлов + 1 моделирование | Руководители департаментов |
| L2 | 34 файла + 3 моделирования | Руководители отделов |
| L3 | ~150 файлов | Сотрудники |

### Требуется смоделировать

| Подразделение | Уровень | Источник |
|---------------|---------|----------|
| Департамент маркетинга и рекламы | L1 | L2: Колдаева, Седых, Четверик |
| Отдел MICE | L2 | L1: Питуркин, L3: Баландина |
| Отдел въездного туризма | L2 | L3: Баландина |
| Группа визового сопровождения | L2 | L3: Анохина, Пономарев |

### Структура департаментов

```
Space Travel
├── MKT — Департамент маркетинга (Моделирование)
├── IT — Департамент IT (Логвинов)
├── FIN — Департамент финансов (Назарикова)
├── OPS — Операционный департамент (Преображенская)
├── REV — Департамент операционных доходов (Баландина)
├── AGN — Департамент развития агентств (Мурадян)
├── INB — Департамент MICE и корп. обслуживания (Питуркин)
├── SRV — Управление клиентского сервиса (Акопова)
└── HR — Департамент HR (Жигалина)
```

### Классификация департаментов

```
┌─────────────────────────────────────────────────────────────┐
│ ГЕНЕРИРУЮЩИЕ ВЫРУЧКУ (Revenue-generating)                   │
│ OPS, AGN, INB, REV — прямое влияние на продажи              │
├─────────────────────────────────────────────────────────────┤
│ ПОДДЕРЖИВАЮЩИЕ ПРОДАЖИ (Revenue-supporting)                 │
│ MKT, SRV — косвенное влияние через лидогенерацию/сервис     │
├─────────────────────────────────────────────────────────────┤
│ ОБЕСПЕЧИВАЮЩИЕ (Cost centers)                               │
│ FIN, IT, HR — внутренние функции                            │
└─────────────────────────────────────────────────────────────┘
```

## ИНФОРМАЦИЯ ДЛЯ ЗАПРОСА У ВЛАДЕЛЬЦА

### Критичная информация (запросить сразу)

---

#### 1. Доступы к САМО и Bitrix24 (для Этапа 3)

**Зачем:** Для анализа IT-ландшафта, текущих интеграций и проектирования автоматизации нужно видеть реальную структуру систем.

**Что запросить:**

| Система | Тип доступа | Что нужно увидеть |
|---------|-------------|-------------------|
| **САМО-тур** | Просмотр (аналитик/менеджер) | Структура заявок, статусы, поля, отчёты |
| **Bitrix24** | Просмотр (аналитик) | CRM-воронки, бизнес-процессы, задачи, интеграции |
| **API-документация** | Ссылка или PDF | Какие данные можно получать/передавать через API |

**Альтернатива:** Если доступ невозможен — скриншоты ключевых экранов и экспорт структуры полей.

---

#### 2. Средние зарплаты по категориям (для Этапа 4)

**Зачем:** Для расчёта ROI (окупаемости) автоматизации. Экономия времени переводится в рубли через стоимость часа сотрудника.

**Что запросить:**

| Категория | Пример должностей | Нужна сумма |
|-----------|-------------------|-------------|
| **Руководители** (L1-L2) | Директора департаментов, руководители отделов | Средняя ЗП gross (до вычета налогов) |
| **Специалисты** | Менеджеры по бронированию, маркетологи, бухгалтеры | Средняя ЗП gross |
| **Линейный персонал** | Ассистенты, операторы, стажёры | Средняя ЗП gross |

**Пример расчёта:**
```
Средняя ЗП специалиста: 80 000 руб/мес
Рабочих часов в месяц: 168 ч
Стоимость часа: 80 000 / 168 = 476 руб/час

Если автоматизация экономит 20 ч/мес:
Ежемесячная экономия: 20 × 476 = 9 520 руб
```

**Формат ответа:** Три числа (руб/мес) — можно округлённо.

**Конфиденциальность:** Данные используются только для расчёта ROI, не публикуются.
