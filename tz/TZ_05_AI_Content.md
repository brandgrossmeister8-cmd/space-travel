## 2. ПРОБЛЕМА (AS-IS)

**Суть проблемы:** Дизайнер создаёт каждый баннер с нуля (2 часа на креатив), а ротацию рекламных объявлений делают вручную, упуская неэффективные кампании.

**Как сейчас работает создание визуалов:**
1. Маркетолог ставит задачу: «Баннер для Турции, лето, семья»
2. Дизайнер ищет референсы (30 мин)
3. Дизайнер ищет фото на стоках (30 мин)
4. Дизайнер создаёт макет в Figma (60 мин)
5. Согласование с маркетологом (2-3 итерации)
6. Итого: 2-3 часа на 1 баннер

**Как сейчас работает управление рекламой:**
1. Маркетолог запускает 5-10 объявлений в кампании
2. Через 2-3 дня смотрит статистику вручную
3. Отключает неэффективные (CTR < 0.5%)
4. Перераспределяет бюджет вручную
5. Часто забывает проверить → бюджет «сливается»

**Боли:**

| Проблема | К чему приводит | Как часто |
|----------|-----------------|-----------|
| Долгое создание визуалов | Задержки кампаний | Каждый день |
| Однотипные креативы | «Баннерная слепота» | 70% материалов |
| Ручная ротация рекламы | Потеря бюджета на слабые объявления | 15-20% бюджета |
| Нет A/B-тестирования | Не знаем, что работает лучше | Постоянно |
| Дизайнер перегружен | Очередь задач 2-3 недели | Постоянно |

**Затраты времени сейчас:**

| Операция | Время | Кто делает |
|----------|-------|------------|
| Создание визуалов (3 баннера/день) | 90 мин/день | Дизайнер |
| Мониторинг и ротация рекламы | 90 мин/день | Маркетолог |
| **ИТОГО** | **180 мин/день** | |

## 3. ЦЕЛЬ (TO-BE)

**Что хотим получить:** ИИ генерирует 10 вариантов визуалов за 5 минут, дизайнер только дорабатывает лучший; реклама автоматически оптимизируется по правилам.

**Как будет работать создание визуалов:**
1. Маркетолог описывает задачу: «Турция, семья, лето, пляж»
2. **ИИ генерирует** 10 вариантов за 5 минут
3. Маркетолог выбирает 2-3 лучших
4. Дизайнер дорабатывает (15-20 мин): логотип, текст, CTA
5. Итого: 30 минут на 3 баннера

**Как будет работать ротация рекламы:**
1. Маркетолог настраивает правила автоуправления
2. **Система автоматически** отключает объявления с CTR < 0.5%
3. **Система автоматически** перераспределяет бюджет на лучшие
4. Маркетолог получает еженедельный отчёт
5. Итого: 30 минут в неделю на контроль

**Целевые показатели:**

| Метрика | Сейчас (AS-IS) | Будет (TO-BE) | Улучшение |
|---------|----------------|---------------|-----------|
| Время на 1 баннер | 90 мин | 10 мин | -89% |
| Баннеров в день | 3 | 10+ | +233% |
| Время на ротацию | 90 мин/день | 15 мин/день | -83% |
| Потери на слабые объявления | 15-20% бюджета | <5% | -75% |
| Время на процессы (общее) | 180 мин/день | 75 мин/день | -58% |

### 7.2. Функции системы

#### F-01: Библиотека промптов для туризма

| Параметр | Значение |
|----------|----------|
| **Приоритет** | Must |
| **Для кого** | Дизайнер, Маркетолог |
| **AUTO-точка** | AUTO-026 |

**Что делает:**
> Как дизайнер, я хочу выбрать готовый промпт для генерации визуала, чтобы не придумывать формулировки каждый раз.

**Структура библиотеки:**

| Категория | Примеры промптов | Количество |
|-----------|------------------|------------|
| **По направлениям** | Турция, Египет, ОАЭ, Мальдивы, Таиланд | 15 |
| **По типу отдыха** | Пляжный, экскурсионный, горнолыжный, круизы | 8 |
| **По аудитории** | Семья с детьми, молодёжь, пенсионеры, романтика | 6 |
| **По сезону** | Лето, зима, Новый год, майские праздники | 5 |
| **По формату** | Баннер 1200×628, сторис 1080×1920, квадрат | 4 |
| **Комбинированные** | Турция + семья + лето + баннер | 15+ |

**Формат промпта (шаблон):**

```
[Направление] travel destination, [тип отдыха] vacation,
[аудитория], [сезон] atmosphere,
[стиль]: vibrant colors, professional photography,
aspect ratio [формат], --v 6 --style raw
```

**Пример готового промпта:**

```
Turkey Antalya beach resort, family vacation with kids,
summer sunny day, happy tourists on beautiful beach,
luxury hotel in background, vibrant tropical colors,
professional travel photography, warm lighting,
aspect ratio 16:9 --v 6 --style raw --no text logos
```

**Критерии приёмки:**
- [ ] Минимум 50 промптов в библиотеке
- [ ] Все направления Space Travel покрыты
- [ ] Промпты протестированы (10+ генераций каждый)
- [ ] Документ доступен всем пользователям

#### F-02: Генерация визуалов через Midjourney

| Параметр | Значение |
|----------|----------|
| **Приоритет** | Must |
| **Для кого** | Дизайнер |
| **AUTO-точка** | AUTO-026 |

**Что делает:**
> Как дизайнер, я хочу генерировать 10 вариантов изображения за 5 минут, чтобы тратить время только на доработку лучшего.

**Сценарий работы:**
1. Маркетолог ставит задачу: «Баннер для Турции, семейный отдых»
2. Дизайнер открывает библиотеку промптов
3. Выбирает/комбинирует подходящий промпт
4. Запускает генерацию в Midjourney (4 варианта)
5. Делает 2-3 вариации лучших (ещё 8 вариантов)
6. Выбирает финальное изображение
7. Дорабатывает в Figma (логотип, текст, CTA)
8. **Итог:** 3 готовых баннера за 30 минут

**Правила работы с Midjourney:**

| Аспект | Правило |
|--------|---------|
| Разрешение | Минимум 1024×1024 (upscale до 2048) |
| Стиль | --style raw для реалистичных фото |
| Без текста | --no text logos watermarks |
| Соотношение | --ar 16:9 для баннеров, --ar 9:16 для сторис |
| Версия | --v 6 (последняя) |

**Альтернативные сценарии:**

| Ситуация | Что делать |
|----------|------------|
| Результат не нравится | Переформулировать промпт, добавить детали |
| Нужен конкретный отель | Добавить в промпт название, найти референс |
| Лица искажены | Использовать --no faces или общие планы |
| Нужен конкретный человек | Использовать фото из стока, не генерировать |

**Критерии приёмки:**
- [ ] Все сотрудники имеют доступ к Midjourney
- [ ] Проведено обучение (4 часа)
- [ ] Созданы первые 20 генераций для реальных задач
- [ ] Качество устраивает маркетинг

#### F-04: Авторотация рекламы в Яндекс.Директ

| Параметр | Значение |
|----------|----------|
| **Приоритет** | Must |
| **Для кого** | PPC-специалист |
| **AUTO-точка** | AUTO-048 |

**Что делает:**
> Как PPC-специалист, я хочу настроить автоправила для отключения слабых объявлений, чтобы не терять бюджет.

**Правила автоуправления:**

| Правило | Условие | Действие | Когда проверять |
|---------|---------|----------|-----------------|
| R-01 | CTR < 0.5% AND показов > 1000 | Отключить объявление | Ежедневно |
| R-02 | CPA > 2× целевого | Снизить ставку на 20% | Ежедневно |
| R-03 | Расход > 3000 ₽ AND конверсий = 0 | Приостановить | Ежедневно |
| R-04 | CTR > 2% AND конверсии > 3 | Увеличить ставку на 15% | Ежедневно |
| R-05 | Позиция < 3 AND CTR < 1% | Поднять ставку на 10% | 2 раза в день |

**Уведомления:**

| Событие | Куда | Кому |
|---------|------|------|
| Объявление отключено | Email | PPC-специалист |
| Кампания приостановлена | Telegram | Руководитель маркетинга |
| Бюджет исчерпан на 80% | Email | PPC-специалист |
| Еженедельный отчёт | Email | Руководитель маркетинга |

**Сценарий работы:**
1. PPC-специалист настраивает автоправила в Яндекс.Директ
2. Система проверяет объявления по расписанию
3. При срабатывании правила — выполняет действие
4. Отправляет уведомление
5. PPC-специалист еженедельно анализирует отчёт
6. **Итог:** бюджет автоматически перераспределяется на эффективные объявления

**Критерии приёмки:**
- [ ] Все 5 правил настроены и работают
- [ ] Уведомления приходят корректно
- [ ] Еженедельный отчёт формируется автоматически
- [ ] Потери на слабых объявлениях снизились
