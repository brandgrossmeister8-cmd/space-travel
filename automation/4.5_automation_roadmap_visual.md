## Критический путь

Критический путь — это последовательность задач, определяющая минимальный срок реализации всего проекта.

```mermaid
flowchart LR
    subgraph Волна0[Волна 0: Подготовка]
        P01[P-01 API САМО] --> TZ01
        P04[P-04 Битрикс24] --> TZ02
        P05[P-05 1С OData] --> TZ07
        P02[P-02 Power BI] --> TZ07
    end

    subgraph Волна1[Волна 1: Quick Wins]
        TZ01[TZ-01 САМО↔Б24]
        TZ02[TZ-02 Шаблоны]
        TZ03[TZ-03 Алерты САМО]
    end

    subgraph Волна2[Волна 2: Core]
        TZ07[TZ-07 BI система] --> DASH01[DASH-01 P&L]
        TZ09[TZ-09 Тайм-лимиты] --> DASH02[DASH-02]
        TZ11[TZ-11 Парсинг цен] --> TZ13
    end

    subgraph Волна3[Волна 3: Advanced]
        TZ12[TZ-12 GDS→1С]
        TZ13[TZ-13 Динам.цены]
    end

    P05 --> TZ12

    style P01 fill:#ff9999
    style P05 fill:#ff9999
    style TZ01 fill:#ff9999
    style TZ07 fill:#ff9999
    style TZ12 fill:#ff9999
    style TZ13 fill:#ff9999
```

**Критические задачи (красные):**
1. **P-01** → Без аудита API САМО невозможны интеграции
2. **P-05** → Без подключения 1С невозможны BI-дашборды и GDS→1С
3. **TZ-01** → Центральная интеграция САМО↔Битрикс24
4. **TZ-07** → Единая BI-система для всей отчётности
5. **TZ-12** → Критический финансовый проект (GDS→1С)
6. **TZ-13** → Стратегическое ценообразование
