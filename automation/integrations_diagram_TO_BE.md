## 2. BP-01: Обработка заявки (Lead-to-Cash) TO-BE

**Изменения vs AS-IS:** мессенджеры зафиксированы в CRM, данные GDS автоматически в 1С, нет ручных переносов.

```
Клиент/Агент
    │
    ├── [Сайт]           → Битрикс24 (API, лид)
    ├── [Телефон/UIS]    → Битрикс24 (API, запись звонка)
    ├── [Max / Telegram] → Битрикс24 (Max API + Telegram Bot API) ⚫ НОВОЕ
    ├── [Email/Outlook]  → Битрикс24 (плагин, письмо-задача) 🔧 УЛУЧШЕНО
    └── [Лендинг/Тильда] → Битрикс24 (webhook, лид)

                           Битрикс24
                              │ API (маршрутизация → нужному менеджеру)
                              ▼
                          САМО-тур
                              │
           ┌──────────────────┼─────────────────────┐
           ▼ API              ▼ XML                  ▼ API (через ИШ)
         GDS              Поставщики              Клиент-банк
    (авиабилеты)            (DMC)                 (оплата)
           │                  │                        │
           └──────────────────┼────────────────────────┘
                              ▼ API (через ИШ) ⚫ НОВОЕ
                       1С Бухгалтерия
                     (авиабилеты, оплаты — всё автоматически)
                              │ OData
                              ▼
                          Power BI
                     (отчётность в реальном времени)

Результат: клиент получает ваучер и договор автоматически после оплаты (AUTO-039)
```

**Устранённые ручные операции (AS-IS → TO-BE):**
- ~~Ручной перенос сообщений из Max / Telegram в CRM~~ → Max API + Telegram Bot API
- ~~Ручной перенос данных GDS в 1С~~ → API через ИШ
- ~~Ручное разнесение платежей из банка~~ → Стандартизированный файловый обмен
- ~~Ручная отправка ваучеров и договоров~~ → AUTO-039

## 3. BP-04: Финансовый цикл TO-BE

**Изменения vs AS-IS:** Invoice Tracker удалён, P&L строится автоматически в Power BI.

```
                ┌─────────────────────┐
                │  Клиент-банки (16)  │
                │  (стандарт. формат) │
                └──────────┬──────────┘
                           │ Файл (унифицированный) 🔧
            ┌──────────────┼──────────────────┐
            ▼              ▼                  ▼
       САМО-тур       1С Бухгалтерия    Счета (1С) ⚫
      (разнесение)    (банк. выписки)   (заменяет Google Таблицы)
            │              │
            └──────────────┤ API (через ИШ) ⚫ НОВОЕ
                           ▼
                      Power BI ⚫ НОВОЕ
                   ┌────────────────────┐
                   │ P&L — авто         │
                   │ Cash Flow — авто   │
                   │ ДЗ/КЗ — авто      │
                   │ Платёж. календарь  │
                   └────────────────────┘

Результат:
- Вместо 12 шагов вручную → кнопка «Обновить» в Power BI
- Invoice Tracker на Google Таблицах → устранён
- Счета создаются в 1С, сразу попадают в платёжный календарь
```

**Устранённые ручные операции:**
- ~~12 шагов ручной обработки P&L в Excel~~ → Power BI + API 1С
- ~~Ручной ввод в Invoice Tracker~~ → Счета в 1С
- ~~Сбои при совместном редактировании Google Таблиц~~ → Устранены

## 4. BP-05: Формирование турпродукта TO-BE

**Изменения vs AS-IS:** автоматизация ценообразования, данные о ценах конкурентов, устранение узкого места в тарификации.

```
Запрос на новый тур
      │
      ▼
  Битрикс24 (задача → группе тарификации)
      │
      ▼
  Группа тарификации
      │ Шаблоны в САМО (AUTO-025) ⚫ НОВОЕ
      ├──── Парсер цен конкурентов (API) ⚫ НОВОЕ ── данные для бенчмарка
      │
      ▼
  САМО-тур

      ├── API (real-time)     → Сайт spacetravel.ru
      ├── API                 → Sletat.ru / Турвизор
      └── API (AUTO-011) ⚫   → Юнисендер (цены в рассылки)

Мониторинг:
  САМО-тур → авторевизия туров без продаж (AUTO-016) 🔧
  САМО-тур → мониторинг сроков действия цен (AUTO-017) 🔧
```

**Устранённые ручные операции:**
- ~~Ручной сбор цен конкурентов~~ → Парсер (AUTO-018)
- ~~Ручной экспорт цен в рассылки~~ → API САМО → Юнисендер
- ~~Ручной ввод из Excel в САМО~~ → Шаблоны в САМО

## 5. BP-06: Клиентская поддержка TO-BE

**Изменения vs AS-IS:** все каналы фиксируются в CRM, отсутствия в одной системе, автомаршрутизация.

```
                   ┌──────────────────────┐
                   │  Входящие обращения  │
                   └──────────┬───────────┘
                              │
    ┌─────────────────────────┼──────────────────────┐
    ▼ API                     ▼ Плагин               ▼ Max/Telegram API ⚫
UIS (звонки)          Outlook (почта)          Max / Telegram
    │  Speech Analytics        │  → задача в Б24      │
    │  → оценка → алерт        │                      │
    └─────────────────────────▼──────────────────────┘
                          Битрикс24
                  (всё фиксируется, ничего не теряется)
                              │
                 ┌────────────┼────────────┐
                 ▼ API        ▼ API ⚫      ▼
             САМО-тур    Моя команда    Power BI
           (данные заявки) (отсутствия) (DASH-16: SLA по каналам)
```

**Устранённые ручные операции:**
- ~~Чаты в Max / Telegram вне CRM~~ → Max API + Telegram Bot API
- ~~Графики отсутствий в Google Таблицах~~ → «Моя команда» + Битрикс24
- ~~Ручная маршрутизация (28 направлений, 15 мин)~~ → Автодиспетчер (AUTO-045, 5 мин)

## 7. HR-процесс TO-BE

**Изменения vs AS-IS:** единая система отсутствий, данные не дублируются.

```
Сотрудник вносит отпуск/отсутствие:
  │
  ▼ [Моя команда] (единственный источник)
  │
  ├── API → Битрикс24 (видно менеджерам) ⚫ НОВОЕ
  └── API → 1С ЗУП (табель, расчёт) 🔧 УЛУЧШЕНО

Приём на работу:
  Битрикс24 (задача HR) → Моя команда (карточка) → 1С ЗУП (расчёт ЗП)

Обучение:
  LMS (iSpring/Teachbase) → HR-аналитика → Power BI DASH-13 ⚫ НОВОЕ
```

## 8. Интеграционная шина — схема работы

```
┌──────────────────────────────────────────────────────────────────┐
│               Интеграционная шина (n8n / Make.com)               │
│                                                                  │
│  Сценарий 1: GDS → 1С                                           │
│  GDS выписывает билет → ИШ забирает PNR → создаёт запись в 1С  │
│                                                                  │
│  Сценарий 2: САМО-тур → 1С (ежедневная синхронизация)           │
│  22:00 → ИШ берёт данные из САМО за день → пишет в 1С           │
│                                                                  │
│  Сценарий 3: Мессенджер → Битрикс24                             │
│  Входящее сообщение Max/Telegram → ИШ → создаёт чат в Б24       │
│  или привязывает к существующей сделке                           │
│                                                                  │
│  Сценарий 4: Алерт тайм-лимита                                  │
│  САМО: тайм-лимит через 24ч → ИШ → уведомление менеджеру в Б24 │
└──────────────────────────────────────────────────────────────────┘
```

**Почему ИШ важна:** без неё каждый из 4 сценариев выше требует разработки с нуля (от 200 до 500 тыс. ₽ каждый). С ИШ IT-отдел настраивает каждый сценарий самостоятельно за 2–5 дней.

## 9. Сравнение диаграмм AS-IS и TO-BE

### Потоки данных AS-IS (схематично)

```
Клиент ──────────────► Битрикс24
                            │ (частичная синхронизация)
                            ▼
                        САМО-тур ──────── [Ручной перенос] ──► 1С
                            │                                    │
                        [Ручной              [Нет интеграции]    │
                         экспорт]                               │
                            ▼                ▼                  │
                        Google Таблицы      GDS ─────────────► │
                        (Invoice Tracker)                       │
                                                                ▼
                                                            Excel P&L
                                                         (12 ручных шагов)
```

### Потоки данных TO-BE (схематично)

```
Клиент ──────────────► Битрикс24 ◄──────── Max / Telegram (API)
        [все каналы]       │ API             Outlook (плагин)
                           │                 UIS Speech Analytics
                           ▼
                        САМО-тур ──── API (ИШ) ──────────► 1С Бухгалтерия
                            │                                    │
                            │ API                                │ OData
                            ▼                                    ▼
                        Юнисендер                            Power BI
                        (цены из САМО)                    (дашборды авто)
                            ▲
                           GDS ──── API (ИШ) ──────────────► 1С (PNR)
```

**Ключевое различие:** в TO-BE нет стрелок с пометкой «ручной». Все потоки данных — API или автоматические файловые обмены.
